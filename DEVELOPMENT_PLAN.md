# Development Goals & Task Breakdown

**Project:** AI for Predicting and Preventing Road Accidents (beginner MVP)

This document turns the idea into **small, executable steps** so you can go from zero to a working demo without getting lost.

---

## 1. Project Goal (MVP Focus)

### Single clear goal

Build a **local Python app** that:

1. Learns from a **tabular accident dataset** (weather, time, place-type style features).  
2. Outputs a **risk level**: Low / Medium / High.  
3. Shows **short safety suggestions** based on that risk and the inputs.  
4. Runs on **your machine** via **CLI or a simple Streamlit screen**—no cloud required.

### What “done” looks like

- [ ] You can run **one command** to train and save a model.  
- [ ] You can run **one command** (or open one app) to get a prediction for sample inputs.  
- [ ] The output is understandable: **risk + recommendation**.  
- [ ] A short **README** (or link to your technical guide) explains how to install and run.  

That is the MVP. Everything else is optional.

---

## 2. Development Philosophy

| Principle | What it means for you |
|-----------|------------------------|
| **Keep it simple** | One dataset, one pipeline (preprocess + model), one saved file (`joblib`). No microservices, no deep learning. |
| **Build incrementally** | After each phase, something should still *run*—even if it only prints a table or a dummy prediction. |
| **Test early** | After training, immediately predict on **one row** you know exists in the data. Fix shape/column errors before building UI. |

### Phase 2: Data Handling ✅ *Complete in repo*

- [x] Write a small script or function to **load the CSV** with Pandas — `utils.load_accidents_csv`, `scripts/run_phase2_eda.py`.  
- [x] Print `shape`, `head()`, `dtypes`, and **missing value counts** (basic EDA) — `utils.print_basic_eda`.  
- [x] Decide **target column** — **`severity_class`** (0 Low / 1 Medium / 2 High); printed when you run the EDA script.  
- [x] Handle missing values — **drop rows** with missing target or features after coerce; invalid severity dropped with a note (see `utils.clean_accidents_df`).  
- [x] Fix obvious formatting issues — **strip** string columns; **numeric coerce** for `hour`, `day_of_week`, target; **clip** hour/day ranges.  

**Exit criteria:** A cleaned DataFrame saved or reproducible in code; you know which column is `y` — **`data/processed/clean_accidents.csv`** (regenerated by the script).

---

### Phase 3: Feature Engineering ✅ *Complete in repo*

- [x] List **5–12 input features** — `hour`, `day_of_week`, `weather`, `area_type`, `traffic_level` (`utils.FEATURE_COLUMNS` + `preprocess.py`).  
- [x] Drop IDs and text you are not modeling in v1 — no ID/long-text columns in the MVP CSV; `X` uses only the five features above.  
- [x] Encode categoricals — `ColumnTransformer` + `OneHotEncoder(handle_unknown="ignore")` + numeric `StandardScaler` in `preprocess.build_preprocessor()`.  
- [x] Build `X` and `y` with **no leakage** — `make_xy()` uses only feature columns; target `severity_class` is separate.  
- [x] Split **train / test** — `preprocess.split_train_test()` with stratify when possible (`test_size=0.25`).  

**Exit criteria:** Training code runs without errors and `X` has consistent column names — run `python scripts/run_phase3_features.py` to verify shapes and feature names.

---

### Phase 4: Model Development ✅ *Complete in repo*

- [x] Train **Logistic Regression** or **Decision Tree** inside the same `Pipeline` as preprocessing — `train.py` (`--model lr` default, `--model dt` optional).  
- [x] Print **accuracy** and **classification report** on the test set — printed after fit.  
- [x] Save the **entire pipeline** with `joblib` — default `models/accident_risk.joblib` (`utils.DEFAULT_MODEL_PATH`).  
- [ ] (Optional) Train the other algorithm and compare — run `python train.py --model dt` if you want a second artifact.  

**Exit criteria:** File on disk; reload + predict works — `train.py` smoke test + `python scripts/verify_model.py`.

### Phase 5: Prediction System ✅ *Complete in repo*

- [x] Write a function to **load** the saved pipeline — `predict.load_pipeline`.  
- [x] Build a **single-row** DataFrame from user inputs — `predict.inputs_to_dataframe` (column order = `FEATURE_COLUMNS`).  
- [x] Map model output (0/1/2) to **Low / Medium / High** — `predict.class_to_risk_level`.  
- [x] Add **recommendation logic** — `predict.build_recommendation` (risk + weather + night + highway/high traffic).  
- [x] Return a **dict** / JSON — `predict.predict_scenario`; CLI prints JSON via `python predict.py ...`.  

**Exit criteria:** Risk + recommendation from inputs — run `python predict.py` with flags (see `predict.py` docstring).

## 7. Common Pitfalls

| Pitfall | Why it hurts | What to do instead |
|---------|----------------|---------------------|
| **Overcomplicating ML** | Weeks lost on “better” models before anything runs | Ship Logistic Regression or shallow Decision Tree first |
| **Bad data handling** | Silent bugs, nonsense accuracy | Always inspect `y.value_counts()` and confusion matrix |
| **Train/predict mismatch** | `predict` crashes or wrong shape | Build prediction row from the **same column list** as `X` |
| **Target leakage** | Fake 99% accuracy | Remove post-accident fields unless that is the actual product goal |
| **Skipping testing** | Broken demo night | After `joblib.dump`, immediately `joblib.load` + one `predict` |
| **Perfect folder structure** | No code written | Create minimal files first; refactor later |

## 8. Starter Priorities (Very Important)

### Do FIRST (Day 1)

1. Working **Python + venv + imports** (pandas, sklearn).  
2. **One CSV on disk** and a 10-line script that loads it and prints `shape`.  
3. **One sentence** defining your label (`y`) for this dataset.  

### Ignore initially (until MVP works)

- Cloud deployment, Docker, databases  
- Multiple datasets merged across cities  
- Deep learning, AutoML, MLOps  
- Fancy frontends, auth, user accounts  
- “Production” monitoring  

Get **train → save → load → predict** on one machine first. Everything else is Phase 2 of your life, not Phase 2 of this week.
